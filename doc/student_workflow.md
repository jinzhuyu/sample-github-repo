# Suggested Student Workflow

## Step 1: Define the project
State the problem, objective, and expected outputs in plain language.

## Step 2: Organize the repository
Create the main folders before writing much code:
- `data/`
- `src/`
- `outputs/`
- `docs/`

## Step 3: Add data documentation early
Even if the dataset is incomplete, describe:
- where it came from,
- what each file contains,
- what is included in the repository, and
- what is omitted.

## Step 4: Build the pipeline in stages
A common order is:
1. preprocessing,
2. modeling,
3. evaluation,
4. visualization.

## Step 5: Keep the code modular
Prefer several focused scripts and reusable functions over one long file.

## Step 6: Update the README continuously
Do not wait until the end. Keep setup steps, folder explanations, and usage instructions current.

## Step 7: Save only important outputs
Store final figures, final tables, and reproducible results in `outputs/`. Avoid clutter.

## Step 8: Prepare the final deliverables
Place the final paper, poster, or slides in `docs/report/`.
